1 Classes & Objects

Classes are blueprints for creating objects in Python, enabling object-oriented programming (OOP) to model real-world entities.

1

Defining a class:

class Dog: def __init__(self, name): self.name = name
2

Creating an object:

my_dog = Dog("Buddy") print(my_dog.name) # Output: Buddy
3

Accessing object attributes:

my_dog.name = "Max" print(my_dog.name) # Output: Max
Did you know?

The __init__ method is a constructor, automatically called when an object is created.

2 Methods & Attributes

Methods are functions defined inside a class, while attributes are variables that belong to an object or class.

1

Defining a method:

class Dog: def __init__(self, name): self.name = name def bark(self): return f"{self.name} says Woof!"
2

Calling a method:

my_dog = Dog("Buddy") print(my_dog.bark()) # Output: Buddy says Woof!
3

Class attribute (shared by all objects):

class Dog: species = "Canis familiaris" # Class attribute def __init__(self, name): self.name = name # Instance attribute dog = Dog("Max") print(dog.species) # Output: Canis familiaris
Key Difference
  • Attributes: Data (e.g., name).
  • Methods: Behavior (e.g., bark).

3 Inheritance & Polymorphism

Inheritance allows a class to inherit properties from another, while polymorphism lets methods behave differently based on the object.

1

Inheritance (child class):

class Animal: def speak(self): return "Sound" class Dog(Animal): def speak(self): return "Woof!"
2

Polymorphism (same method, different behavior):

dog = Dog() animal = Animal() print(dog.speak()) # Output: Woof! print(animal.speak()) # Output: Sound
3

Using parent method:

class Cat(Animal): pass cat = Cat() print(cat.speak()) # Output: Sound (inherited)
Pro Tip

Use super() to call a parent class’s method from a child class.

4 Encapsulation

Encapsulation hides an object’s internal details and exposes only what’s necessary, often using private attributes.

1

Basic encapsulation with private attributes:

class Car: def __init__(self): self.__speed = 0 # Private attribute def set_speed(self, speed): self.__speed = speed def get_speed(self): return self.__speed
2

Using methods to access private data:

my_car = Car() my_car.set_speed(100) print(my_car.get_speed()) # Output: 100
3

Direct access fails:

# print(my_car.__speed) # AttributeError
Did you know?

Double underscores (__) trigger name mangling in Python to enforce privacy.

5 Abstraction

Abstraction hides complex implementation details and shows only the essential features, often using abstract base classes.

1

Defining an abstract base class:

from abc import ABC, abstractmethod class Shape(ABC): @abstractmethod def area(self): pass
2

Implementing a concrete class:

class Circle(Shape): def __init__(self, radius): self.radius = radius def area(self): return 3.14 * self.radius ** 2
3

Using the abstraction:

circle = Circle(5) print(circle.area()) # Output: 78.5
Pro Tip

You can’t instantiate an abstract class directly; it must be subclassed with all abstract methods implemented.
